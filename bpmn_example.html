Пример описания Бизнес-требований

Продукт

Результатом процесса является оформление и подписания сторонами(Физическое лицо и Фонд) договора ПДС. Данные, предоставленные потенциальным клиентом, проверяются с помощью сервисов, которые предоставляет внешний подрядчик. Подписание договора и комплекта сопутствующих документов осуществляется в соответствии со статьей 9 федерального закона 63-ФЗ «Об электронной подписи». Клиент получает комплект документов, подписанный фондом, и ссылку на выполнение оплаты по внесению первоначального взноса. По сравнению с имеющимся вариантом оформления ПДС, после реализации доработок, договор ПДС смогут оформить и те клиенты, которым не доступна аутентификация с помощью Госуслуг.

Перечень пользовательских историй(пример)

US1.
Пользователь с ролью «Все пользователи», в процессе регистрации договора ПДС, до момента формирования документов по договору, должен иметь возможность аутентифицироваться на Госуслугах и заполнить анкету на заключение договора ПДС данными с Госуслуг. После такой аутентификации он получает роль «Авторизованные через ГосУслуги». Заполненные данные, в этом случае, не подлежат редактированию. Для того чтобы сократить время оформления договора ПДС за счет автоматического заполнения полей анкеты и пропуска этапа проверки клиентских данных(Фин. Мониторинг, СФР, ФНС

US2.
Пользователь с ролью «Все пользователи», должен иметь возможность аутентифицироваться на сайте с помощью номера договора ПДС и sms-кода, который отправляется на телефон указанный в договоре. Для того чтобы он мог посмотреть уведомления по своему договору на сайте НПФ.

US3 Пользователь с ролью «Все пользователи», при регистрации договора ПДС, до момента формирования документов по договору, должен иметь возможность редактировать анкетные данные и желаемые параметры договора. Для того чтобы клиент и Фонд могли заключить корректный договор, наиболее полно отвечающий их интересам.

US4.1 Пользователь с ролью «Все пользователи», при регистрации договора ПДС, после активации формирования документов по договору, должен иметь возможность пройти проверки по ПОД ФТ: Проверка осуществляется по ...... , и считается НЕ пройденной в следующих случаях: если Клиент ...... Проверки выполняются по спискам, которые предоставляют ....... Анкетные данные, не прошедшие проверку по ПОД ФТ, не позволят заключить договор. Для того чтобы НПФ мог избежать рисков связанных с требованиями законодательства РФ в части противодействия отмыванию денег и финансирования терроризма.
